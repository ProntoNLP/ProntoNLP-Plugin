# Evaluations — pronto-trends

Evaluation criteria for assessing the quality of keyword and topic trend responses generated by this skill.

## Evaluation Dimensions

### Tool Selection
- Was `getTrends` used only when the user explicitly said "trends" or "trending"?
- Was `getMindMap` used for relationship/concept map requests?
- Was `getTermHeatmap` used for term frequency requests?
- Were movers tools (`getTopMovers`, `getDeepResearchStockAverage`) correctly NOT used here?

### Execution Correctness
- Was `getTrends` sorted by `score` descending with limit 20?
- Was the correct scope applied (companyName or sector) to `getTrends`?
- Was `getMindMap` output converted to a readable nested structure?
- Was `getTermHeatmap` output sorted by frequency descending?

### Trends Response Quality
- Does the response include a table with Score, Hits, % Change, Direction?
- Are RISING / DECLINING labels correct based on % change sign?
- Are the top 3 fastest-rising themes explicitly called out with brief explanations?
- Are notable declining themes included?
- Are at least 10 topics presented?

### Mind Map Quality
- Is the mind map rendered as a nested markdown list (not a raw JSON dump)?
- Are related sub-themes grouped under clear parent nodes?

### Term Heatmap Quality
- Is the output presented as Term | Company/Quarter | Frequency?
- Is it sorted by frequency descending?
- Are frequency spikes flagged?

### Citations
- Does every factual statement have a `[N][N]` citation link?
- Are citation links pointing to `https://dev.prontonlp.com/#/ref/<ID>`?
- Are numbers sequential and correct?

## Scoring

Each dimension is scored 1–5:
- 5: Exceptional — exceeds expectations
- 4: Good — meets all requirements
- 3: Adequate — minor gaps
- 2: Below expectations — significant gaps
- 1: Failing — major issues

Overall score = average of all dimensions.
